# Section Writing Guidance — NSO Nepal

What to include in each common section of an official report, plus statistical writing rules.
Load this file in Stage 5 when planning the content of a section.

---

## What Each Section Must Do

### Cover Page
Include: report title, subtitle (if any), publishing institution (National Statistics Office, Government of Nepal), the period covered, edition number (for recurring reports), and the Government of Nepal emblem.

### Foreword / Preface
- Written by the Director General or a senior official
- Typically 1–2 paragraphs
- Contextualises the importance of the report
- Thanks contributors and funders
- Does NOT repeat findings — that is the Executive Summary's job

### Executive Summary
- The most-read section of any long report — write it LAST
- Must stand alone: a reader who reads only this should understand the main message
- Short: 1–2 pages for a major report; 3–5 lines for a bulletin
- Must include: purpose, key findings, and key recommendations (if applicable)
- Paragraph structure:
  - Para 1: Purpose and scope of the report (2–3 sentences)
  - Para 2–3: The most important 3–5 findings (one finding per paragraph)
  - Final para: Recommendations or next steps (if applicable)

### Introduction / Background
Sets the scene. Must answer:
- Why was this report written?
- What statistical question or policy need does it address?
- What is the time period, geography, and population covered?
- How does it relate to previous reports in the same series?

Paragraph structure:
- Para 1: Context — the broader situation or policy background
- Para 2: The specific purpose and objectives of this report
- Para 3: Scope, coverage, and reference period
- Para 4 (optional): How to use/read the report

### Methodology / Data Sources
Makes findings credible and replicable. Include:
- Data source(s): administrative records, survey, census, satellite data, etc.
- Sample design (if a survey): sampling frame, sample size, stratification method
- Data collection period and methods (face-to-face interview, phone, register, etc.)
- Processing steps: editing rules, imputation, weighting
- Key limitations and caveats

**For a press release:** reduce methodology to one sentence.
Example: *"Data is sourced from the Nepal Consumer Price Index survey, conducted across five ecological zones covering all 77 districts."*

### Findings / Results
The heart of the report. For each finding:
1. State the number/fact itself — clearly and first
2. Provide a comparison (to previous period, national target, or international benchmark)
3. Give a brief interpretation — what does this change mean in practical terms?
4. Reference the supporting table or chart

**Structure options:**
- For large reports: thematic chapters (e.g., Chapter 3: Employment; Chapter 4: Wages)
- For bulletins: indicator sections (e.g., 3.1 Overall CPI; 3.2 Food Inflation)

### Analysis / Discussion
Goes beyond presenting data to explaining what it means:
- What patterns emerge across findings?
- What are the possible explanations for the trends?
- Are there notable differences by region, gender, age group, or income level?
- How does this compare to other countries or to the previous survey round?

Keep this section OBJECTIVE — official statistical analysis is descriptive and comparative, not opinionated. Do not use phrases like "the government should" unless the brief explicitly calls for recommendations.

### Conclusions
- Summarise what the data SHOWS — do not introduce new information
- One paragraph per major finding or theme
- Written in simple, declarative sentences
- Should link back to the objectives stated in the Introduction

### Recommendations
- Include only if the report brief calls for them (policy briefs always do; bulletins usually do not)
- Be specific: name the action, the responsible party, and the timeframe if known
- Base each recommendation directly on a finding — never recommend something not supported by the data

### Annexes
House technical material that would slow down the main text:
- Full data tables (if main text only has summary tables)
- Survey questionnaire
- Sampling frame details and maps
- Definitions and classification systems (NSCO, NACE, etc.)
- List of enumeration areas

## Statistical Writing Rules

Apply these rules in every section where numbers appear.

### Rule 1: Let the number lead, then explain it
- **Weak:** "There has been a significant upward trend in price levels."
- **Strong:** "Consumer prices rose by 6.2% year-on-year in Baisakh 2082, up from 5.8% in Chaitra 2081."

Always give the actual number first. Avoid vague adjectives like "significant", "high", "considerable", or "dramatic" unless followed immediately by a number.

### Rule 2: Anchor every number in three dimensions
Whenever you state a number, always specify:
- **The indicator:** Is it the unemployment *rate* or the *number* of unemployed persons?
- **The reference period:** Month, quarter, fiscal year — and always specify whether Nepali calendar (e.g., Baisakh 2082) or English calendar (e.g., April/May 2025)
- **The population:** All persons aged 15+? Urban households only? Nepal as a whole?

### Rule 3: Percentage vs. Percentage Points — never confuse these
This is one of the most common errors in statistical writing.
- If inflation rises from **5% to 6%**, it has risen by **1 percentage point** — NOT "by 1%"
- A 1% *increase in a rate* of 5% would bring it only to **5.05%**
- Always write: "rose by X percentage points" when comparing two rates

### Rule 4: Tables and Figures — every one must have five elements
1. **Number** — Table 1, Figure 3, etc. (number sequentially through the document)
2. **Title** — fully describes what it shows (someone should understand without reading surrounding text)
3. **Unit of measurement** — in the column header or subtitle (%, NPR, persons, kg, etc.)
4. **Source line** — at the bottom: "Source: NSO Nepal, [Survey Name], [Year]"
5. **Notes** — any footnotes, rounding rules, exclusions, or definitions

Keep tables near the text that references them — do not put all tables in annexes.

### Rule 5: Round numbers intelligently
- In headlines and opening paragraphs: round to a readable figure ("about 1.2 million persons")
- In body text and tables: give the precise figure (1,213,447 persons)
- Never drop the precise figure entirely — always include it somewhere

### Rule 6: Avoid relative comparisons without anchors
- **Avoid:** "Poverty fell by 30%."
- **Better:** "The poverty rate fell from 25.2% in 2010/11 to 17.4% in 2022/23, a decline of 7.8 percentage points."

### Rule 7: Qualify uncertainty honestly
- If estimates have margins of error, state them: "The unemployment rate was estimated at 11.4% (±1.2 percentage points)"
- If data from a district or group is unreliable due to small sample size, flag it clearly
- Transparency about data limitations is a mark of credibility — not a weakness

### Rule 8: Define technical terms on first use
Any term that a non-statistician might not know must be defined in brackets the first time it appears.
- Example: "The labour force participation rate (the share of the working-age population who are either employed or actively seeking work) stood at 39.2% in 2081."

## Language and Tone Guide

| Avoid | Use Instead |
|-------|------------|
| "Significant increase" | "Increased by 3.4 percentage points" |
| "The data shows that clearly..." | "The data shows that..." |
| "It is interesting to note..." | Just state the finding |
| "A large number of people..." | "An estimated 2.3 million persons..." |
| "Due to the fact that..." | "Because..." |
| "In order to..." | "To..." |
| "It should be noted that..." | Just say it |
| "Prior to..." | "Before..." |
| "Utilise" | "Use" |
| "Facilitate" | "Help" or "support" |

**Sentence length:** Keep most sentences under 25 words. For complex data findings, use two shorter sentences instead of one long one.

**Paragraph length:** Each paragraph should make one main point. Aim for 3–5 sentences. Do not let paragraphs run longer than 7–8 sentences.

---

## Quick Checklist Before Submitting a Draft Section

- [ ] Does every number have a reference period, population, and unit?
- [ ] Are percentage point changes written correctly (not as "percent")?
- [ ] Does each table/figure have a number, title, unit, source, and notes?
- [ ] Are all technical terms defined on first use?
- [ ] Does every paragraph have one clear main point?
- [ ] Is the tone objective — no opinions, no vague adjectives without numbers?
- [ ] Would an undergraduate student understand this section?
